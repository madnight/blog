Monad
A Monoid in the Category of Endofunctors
Posted on Jul 25 2023 ~ 4 min read
#category theory 


A Monad is a triple ( T , η , μ ) (T, \eta, \mu) (T,η,μ) where:

  • T : C → C T: \mathcal{C} \rightarrow \mathcal{C} T:C→C is an endofunctor
  • η : 1 C → T \eta: 1_\mathcal{C} \rightarrow T η:1C​→T is a natural transformation (the unit)
  • μ : T T → T \mu: TT \rightarrow T μ:TT→T is another natural transformation (the multiplication)

These must satisfy the following coherence conditions, known as the Monad laws:

  • μ ∘ T μ = μ ∘ μ T \mu \circ T\mu = \mu \circ \mu T μ∘Tμ=μ∘μT (associativity)
  • μ ∘ T η = μ ∘ η T = 1 T \mu \circ T\eta = \mu \circ \eta T = 1_T μ∘Tη=μ∘ηT=1T​ (left and right identity)

This means that for any object X X X in C \mathcal{C} C, we have:

  • μ X ∘ T ( μ X ) = μ X ∘ μ T ( X ) \mu_X \circ T(\mu_X) = \mu_X \circ \mu_{T(X)} μX​∘T(μX​)=μX​∘μT(X)​ (associativity)
  • μ X ∘ T ( η X ) = μ X ∘ η T ( X ) = i d T ( X ) \mu_X \circ T(\eta_X) = \mu_X \circ \eta_{T(X)} = id_{T(X)} μX​∘T(ηX​)=μX​∘ηT(X)​=idT(X)​ (left and right unit laws)

We can rephrase these conditions using the subsequent commutative diagrams:

\begin{xy} \xymatrix{ TTT \ar[d]_{\mu T} \ar[r]^{T\mu} & TT \ar[d]^{\mu} \\ TT \ar[r]_{\mu} & T } \end{xy}
\begin{xy} \xymatrix{ T \ar[d]_{\eta T} \ar[r]^{T\eta} \ar@{=}[dr] & TT \ar[d]^{\mu} \\ TT \ar[r]_{\mu} & T } \end{xy}

An application of this concept is that monads provide a way to express computations (in terms of morphisms) that include additional structure or side-effects (captured by the endofunctor T T T) in such a way that these computations can be chained together (via the μ \mu μ natural transformation) and lifted over the monadic structure (via the η \eta η natural transformation), and they do so in a way that is consistent (respecting the associativity and unit laws).

Example

The Monad, by definition, requires us to implement two functions: the unit, which is called return in Haskell, where we just have to lift a value into the Monad (e.g., put a value into a list), and the multiplication >> (sequence) or >>= (bind). The sequence can be used if you are only interested in collapsing the structure (TT -> T). The bind function can be used if you need to operate on the lifted value before collapsing.
